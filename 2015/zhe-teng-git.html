折腾Git！

技术 ?? Comments 周四 05 十一月 2015

配置Terminal中的Git

这文章就是为了测试

折腾了这么久的命令行Git，现在始终得测试一下是否可用。

我成功啦！

终于学会了怎么去配置本地的Git仓库，怎样去download远程仓库的内容到本地制定目录。可能各位大神不觉得我这有什么问题，可之前我是这样操作的：

#找到一个Git地址
#例如 https://github.com/xfhrv/xfhrv.github.io.git
#然后随便找个目录，例如 $HOME 目录下

$ cd ~

#然后直接git clone出来
~$ git clone https://github.com/xfhrv/xfhrv.github.io.git

#本地就会多一个名为 xfhrv.github.io 的文件夹，里面装的文件
~$ ls
xfhrv.github.io

可是我用的纸小墨InkPaper是把最新的文件生成到一个名为Public的文件夹里，关键是网站程序自带的./ink publish发布功能我没法用！因为我不会！Handle Book里边也没讲！今晚我就是在折腾这些东西！

How to do

既然我已经折腾出来了方法，那我还是针对小白写一个Step by step吧。

这里我假设大家跟我一样，Github里已经有远程仓库了，而且原创仓库里有内容了，只是需要把文件弄到本地来修改，经过简单的配置就可以轻松使用git push内容到远程仓库。

Step 1 （建立）进入文件夹

#如果没有文件夹，先建立，如果已经有了，直接进入

$ mkdir test
$ cd test
test$

Step 2 初始化

test$ git init
Initialized empty Git repository in /test/.git/
test$

Step 3 仓库别名

#这一步我的理解就是给冗长的Git远程仓库地址设置一个本地好用的别名
#一般都是设置为 origin ，你也可以改成别的

test$ git remote add origin git@github.com:username/project.git
test$

Step 4 把远程仓库的内容“拉”回来

#这句话的意思就是把git远程的origin仓库里的master分支的内容给我拉回来
test$ git pull origin master

#然后git分分钟开始当搬运工
remote: Counting objects: 3, done.
remote: Compressing objects: 100% (3/3), done.
remote: Total 3 (delta 99), reused 0 (delta 0), pack-reused 90
Receiving objects: 100% (3/3), 2.34 MiB | 52.00 KiB/s, done.
Resolving deltas: 100% (106/106), done.
From github.com:username/project
 * branch            master     -> FETCH_HEAD
 * [new branch]      master     -> origin/master

#运回来3个文件 
test$ ls
CNAME   index.html  resume.html

东西都运回来了，现在这个test文件夹就是你的本地仓库了，如果修改了仓库里的文件，要运走怎么办？

更简单好吗！

Step 1 让git去查修改的东西有哪些

#简单，就一条命令
test$ git add .
test$
#没有任何提示，git默默在后台把事儿给做了

#如果你想查看明细
test$ git status

On branch master
Changes to be committed:
  (use "git reset HEAD <file>..." to unstage)

deleted:    test

test$
#这里看见删除了一个名叫test的文件

Step 2 Commit提交并说明

#你修改过东西，是不是得自己确认一次？是不是得备注修改过什么？
#那就 commit 提交

test$ git commit -m "这里就写注释的内容"
[master e481e2d] for test
1 file changed, 1 deletion(-)
delete mode 100644 test
test$ 
＃git会提示一些基础信息

Step 3 最后把东西“推”上去

#这个也简单
#让git把“货”推到别名origin仓库的master分支去
test$ git push origin master

#然后会显示一些传输的过程信息
Counting objects: 1, done.
Delta compression using up to 4 threads.
Compressing objects: 100% (1/1), done.
Writing objects: 100% (1/1), 213 bytes | 0 bytes/s, done.
Total 1 (delta 1), reused 0 (delta 0)
To git@github.com:username/project.git
    caae58f..e481e2d  master -> master
test$

以上的内容简单吧！

内容是简单，可是还是耽搁了我大好晚上的不少时间，本着每天学习一点点的心态，抱着颗石头也得嚼碎咽下去的信念，继续努力吧，少年！

在这里，我必须得感谢前辈们，没有你们的教程，我也没法那么快弄明白。
