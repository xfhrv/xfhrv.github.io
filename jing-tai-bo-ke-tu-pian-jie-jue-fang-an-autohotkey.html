静态博客图片解决方案——AutoHotkey

Posted on 周一 06 三月 2017 in 技巧

静态博客搭建不容易，使用起来不容易，坚持使用更不容易。 但是，请不要轻易放弃。 今天我就遇到个特别麻烦的问题：给文章添加图片。 是的，静态博客管理图片就是那么麻烦。

事情是这样的，我今晚的目标本是写一片管理Civil3D地形处理的文章，属于step by step的形式。既然是这样的文章，那光用枯燥的文字很难去描述操作步奏咯，那就添加软件操作的截图。刚准备动手，就发现OMG，用Markdown写博客添加图片好麻烦啊！

当时我的内心是这样的：

然后我就开启了百度模式，在网上疯狂地搜索，看看大神们、前辈们是怎样去解决这个问题的。 诶？发现一个好东西：MWeb

刚一点开，我又崩溃了：这么好的东西，只支持Mac平台。 对我们Win环境的盆友们太不友善了啊！ （其实我还是有Mac的，只是懒得再去折腾多平台部署Pelican来写文了）

ok，那我继续找，思路就往workflow、IFTTT、Automator这个方向去了。

然后我就发现了一个神器！！

AutoHotkey！！

参考了网上两篇文章（文章1、文章2），down（AutoHotkey官网）了程序，就开始自己动手研究这东西了。

经过我的研究，实现了这样的功能：

1、选中一张你想添加到文章里的图片

2、按下某个类似于Ctrl+C的快捷键（自定义任意组合）

3、图片自动跑到你博客程序的images文件夹里去了

3、图片自己给自己设置了一个永远不会重复冲突的名字

3、图片把自己的路径放进了剪贴板

4、你在你写的文章里Ctrl+V粘贴即可

是不是很方便！！！

为了造福广大人民群众，我把AutoHotkey的脚本代码放上来

/*

作者：徐费皓苒  xfhrv.cn

说明：此脚本实现了复制选中的图片，自动copy到指定目录，且重命名为当前时间，并将文件名copy到剪贴板备用

用法：运行脚本，选中需要copy的图片，按下Ctrl+Shift+C即可

*/


^+c::

send, ^c

clipwait

filename = %clipboard% ;copy剪贴板的完整文件路径、名称

subpart = \

StringGetPos, index, filename, %subpart%, R

if index >= 0
{

  name := SubStr(filename, index+2)
  path := SubStr(filename, 1,index+1)
  ext := SubStr(filename, -3)

}

newname = %A_Now%%ext%

FileCopy, %path%%name%, C:\Users\xfhrv\Desktop\blog\content\images\%newname%

clipboard = ![](/images/%newname%)

return
