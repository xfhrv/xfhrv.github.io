Dynamo杂症：独立打开程序无3D界面、无法读取Excel文件

Posted on 周四 30 三月 2017 in 技术

Dynamo国内资源太少，很多疑难杂症要么去官方论坛找，要么自己研究，我把我遇到的问题和解决方案放在这里供大家参考，希望可以给您带来帮助！

今天我分享一下我遇到的两个问题：

  1. 独立打开Dynamo程序无法显示3D界面，3D Watch无法预览；
  2. 读取Excel文件无法读取，提示"警告:一个或多个输入类型不匹配。找不到采用类型 (string,string,bool) 参数的 ReadFromFile 的版本"。

ok，下面是解决方案。

问题一：独立打开Dynamo程序无法显示3D界面，3D Watch无法预览

有些朋友会遇到这样一个问题：通过Revit 2017里的按钮点击打开Dynamo，它一切正常；如果不打开Revit，单独开启Dynamo，它的3D界面就一片空白，3D Watch也一片空白，什么也不能看，什么也不能做。

通过Revit打开正常：

问题剖析：单独开启Dynamo时没有打开显卡的3D加速，问题多出现与双显卡的机器。

解决方案：打开显卡的3D加速功能。

我的电脑是Intel核心集成显卡+NVIDIA独立显卡，默认打开Dynamo是使用的集成显卡，没有开启3D加速功能，所以导致Dynamo会这么神经，什么也不显示。解决起来也非常简单，把Dynamo添加到独立显卡的程序列表里就可以了。

问题二：读取Excel文件无法读取，提示警告信息

这个问题是这样的，当你用 File Path -> Excel.ReadFromFile 的时候，会发现文件无法读取，就像这样：

这个问题也会给大家带来一些困惑。

问题剖析：我查了一下Dynamo官方论坛的，有人分析是因为路径地址存在空格，导致路径读取不正确。

解决方案：“加工”一下路径，使用 "%20"替换" "

  1. 添加一个 Code Block 填入代码 String.Replace(parsepath,"%20", " ");
  2. 添加一个 File.FromPath
  3. 将File Path -> Code Block -> File.FromPath ->Excel.ReadFromFile串起来
  4. 搞定！
